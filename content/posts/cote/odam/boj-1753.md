## 문제 요약

방향 그래프에서 주어진 시작점부터 다른 모든 정점까지의 최단 경로를 구하는 문제.
정점 수 V (≤20000), 간선 수 E (≤300000), 가중치는 양의 정수.

## 내 첫 접근 (오답)

BFS로 접근했다. 간단히 큐에 `(거리, 노드)` 를 넣고 탐색했는데 출력이 계속 틀렸다.

## 오답 이유

BFS는 **가중치가 없는(균일한)** 그래프에서만 최단경로를 보장한다.
가중치가 있으면 짧은 간선 하나가 긴 간선 여러 개보다 나쁜 경로일 수 있다.

## 정답 접근법

**다익스트라 알고리즘** 사용.

1. 거리 배열 `dist`를 INF로 초기화, 시작점만 0
2. `heapq`(최소 힙)에 `(0, start)` 삽입
3. 힙에서 꺼낼 때 `dist[v] < 현재 거리`면 스킵 (이미 더 짧은 경로 확정)
4. 인접 노드 거리 갱신 후 힙 push

## 핵심 포인트

- `heapq.heappush(heap, (cost, node))` — 비용 기준 자동 정렬
- 방문 체크 배열 대신 **거리 비교**로 중복 처리
- 도달 불가 정점은 `"INF"` 출력

## 풀이 코드

```python
import heapq
import sys
input = sys.stdin.readline
INF = float('inf')

V, E = map(int, input().split())
K = int(input())

graph = [[] for _ in range(V + 1)]
for _ in range(E):
    u, v, w = map(int, input().split())
    graph[u].append((v, w))

dist = [INF] * (V + 1)
dist[K] = 0
heap = [(0, K)]

while heap:
    cost, u = heapq.heappop(heap)
    if dist[u] < cost:
        continue
    for v, w in graph[u]:
        if dist[u] + w < dist[v]:
            dist[v] = dist[u] + w
            heapq.heappush(heap, (dist[v], v))

for i in range(1, V + 1):
    print(dist[i] if dist[i] != INF else "INF")
```
